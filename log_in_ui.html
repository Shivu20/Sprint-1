Log In - UI

Description

As a customer, I should be able to log in to the application using the username and password.

Acceptance Criteria

The login page should have the following details:

  • Username and Password fields
  • Error messages for invalid input
  • Successful login should redirect to a dashboard with role-specific options
User ID must be between 5 and 20 characters.
Password must contain at least one uppercase letter, one lowercase letter, and one special character.
